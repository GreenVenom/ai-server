## 📐 Versioning Policy

The platform follows Semantic Versioning.

Major versions represent production releases.

Minor versions correspond to milestone completion.

Patch versions are documentation, maintenance, and bug fixes.

- Documentation review
- Architecture review
- ADR review
- Successful health checks
- Updated benchmarks (if applicable)
- Git tag
- Milestone documentation
