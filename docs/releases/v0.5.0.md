# v0.5.0 — M05 Obsidian Integration

## Summary

v0.5.0 completes M05 and introduces controlled, local-first retrieval over an approved Obsidian vault.

## Highlights

- Added a secure read-only Obsidian vault mirror.
- Added full and incremental Markdown ingestion.
- Added local embeddings through `nomic-embed-text:latest`.
- Added the production Qdrant collection `obsidian_chunks_v1`.
- Added deterministic manifests and deletion safety.
- Added semantic retrieval with source path and heading metadata.
- Added the OpenClaw `obsidian_search` tool.
- Added hourly synchronization and indexing through a LaunchAgent.
- Added health, status, backup, checksum validation, and operational runbooks.

## Verified production baseline

```text
Vault ID             personal-knowledge
Documents            7
Production chunks    176
Collection points    184 including fixture data
Health checks        55 passed, 0 warnings, 0 failed
```

## Upgrade notes

- Set the platform version to `v0.5.0` in `scripts/config/platform.conf`.
- Install and build the OpenClaw plugin.
- Install the LaunchAgent plist into `~/Library/LaunchAgents`.
- Register the vault configuration and read-only Git checkout.
- Run the full production index once before enabling scheduled incremental runs.

## Known follow-up

Inline `data:image/...` content in imported ReadItLater notes should be stripped in a future parser improvement to reduce retrieval noise.

## Next steps

M06 will expose approved platform capabilities through narrow MCP interfaces while reusing the M05 retrieval boundary.
